# 제품 요구사항 정의서 (PRD): AI 리터러시 챌린지 『진짜? 가짜?』

- **프로젝트명:** AI 리터러시 챌린지: 『진짜? 가짜?』
- **버전:** 1.2 (Complete)
- **작성일:** 2025년 10월 12일

## 1. 개요 (Overview)

본 문서는 생성형 AI에 대한 학생들의 비판적 사고 및 기초 소양(Literacy) 함양을 위한 인터랙티브 웹앱 『진짜? 가짜?』의 제품 요구사항을 정의합니다. 사용자는 이미지와 텍스트 두 가지 유형의 챌린지를 통해 AI 생성물의 특징과 한계를 직접 체험하고, 합리적으로 판단하는 능력을 기르게 됩니다. 본 웹앱은 데스크톱 및 모바일 환경에 최적화된 반응형 웹으로 개발됩니다.

## 2. 목표 (Goals & Objectives)

- **제품 목표:** 사용자가 AI 생성 콘텐츠와 실제 콘텐츠를 구별하는 경험을 통해 AI 기술에 대한 균형 잡힌 시각을 갖도록 돕는다.
- **학습 목표:**
    - AI 생성물의 정교함(가치)과 기술적 허점(한계)을 분석하는 능력을 기른다.
    - AI 기술이 만들어내는 사회적 이슈(e.g., 딥페이크, 가짜뉴스)에 대한 문제의식을 갖는다.
    - 디지털 콘텐츠를 비판적으로 수용하고 판단하는 자기주도적 학습 태도를 형성한다.

## 3. 사용자 페르소나 (User Persona)

- **이름:** 김인공
- **신분:** 고등학생
- **특징:**
    - 다양한 소셜 미디어와 온라인 플랫폼을 통해 AI가 만든 이미지나 텍스트를 무분별하게 접하고 있다.
    - AI 기술에 대해 막연한 호기심과 흥미를 가지고 있지만, 그 원리나 잠재적 위험성에 대해서는 깊이 생각해 본 적이 없다.
    - 수동적인 정보 습득보다 직접 참여하고 경험하는 방식의 학습을 선호한다.
    - 주로 스마트폰을 사용하여 학습 자료에 접근한다.

## 4. 제품 기능 요구사항 (Product Features & Requirements)

### [Screen 1.0] 시작 및 유형 선택 화면

- **UI/UX:**
    - 중앙에 앱의 제목 『진짜? 가짜?』와 "AI가 만든 콘텐츠, 구별할 수 있나요?" 와 같은 부제를 표시한다.
    - 사용자가 선택할 수 있는 두 개의 큰 카드/버튼을 나란히 배치한다.
        - **[이미지 챌린지]**: "AI vs 사람: 진짜 사진을 찾아라!"
        - **[텍스트 챌린지]**: "AI vs 사람: 진짜 글을 찾아라!"
- **반응형 디자인:**
    - **데스크톱:** 두 카드가 가로로 나란히 표시된다.
    - **모바일:** 두 카드가 세로로 쌓여 표시된다.

### [Screen 2.0] 챌린지 진행 화면

- **공통 기능:**
    - 화면 상단에 현재 진행 중인 레벨(Level 1, 2, 3)을 표시한다.
    - 콘텐츠(이미지 또는 텍스트) 제시 후, 하단에 `[AI 생성]`, `[실제]` 두 개의 선택 버튼을 제공한다.
    - **힌트 기능 (Level 2):** '힌트 보기' 버튼을 제공하며, 클릭 시 모달창이나 툴팁으로 결정적인 단서를 암시하는 텍스트를 보여준다. (e.g., "이미지의 배경을 자세히 살펴보세요.")

- **2.1 이미지 챌린지 상세:**
    - **진행 방식:** 하나의 이미지를 제시하고, 이 이미지가 'AI 생성'인지 '실제 사진'인지를 판별하게 합니다. 사용자는 주어진 이미지가 AI가 만든 것인지, 실제 촬영된 사진인지를 선택해야 합니다.
    - **Level 1 (튜토리얼):** 6개의 손가락, 부자연스럽게 녹아내리는 배경 등 누가 봐도 식별 가능한 오류가 포함된 AI 실사 이미지를 제시한다. (실제 사진과 함께 제시)
    - **Level 2 (훈련):** 빛의 방향과 그림자의 불일치, 어색한 질감 표현, 액세서리의 비대칭 등 미묘한 오류가 포함된 AI 이미지를 제시한다. (실제 사진과 함께 제시)
    - **Level 3 (실전):** 기술적 결함은 거의 없으나, 물리적으로 촬영이 불가능하거나 현실성이 극히 떨어지는 극적인 상황의 AI 이미지를 제시한다. (실제 사진과 함께 제시)

- **2.2 텍스트 챌린지 상세:**
    - **주제:** 모든 텍스트는 'AI 리터러시'와 관련된 내용으로 구성한다.
    - **Level 1 (튜토리얼):** 2~3문장의 짧은 글에서 명백히 앞뒤가 맞지 않는 문장을 포함한다. (e.g., "AI는 인간의 감정을 완벽히 이해합니다. 따라서 AI는 감정을 느끼지 못합니다.")
    - **Level 2 (훈련):** 한 문단 정도의 글에서, 그럴듯하지만 자세히 읽으면 논리적 비약이나 모순이 있는 부분을 포함한다.
    - **Level 3 (실전):** 두 문단 이상의 긴 글에서, 전체적인 주제와 미묘하게 벗어나거나 일관성이 결여된 주장을 포함하여 맥락적 추론 능력을 요구한다.

- **2.3 피드백 기능 (선택 직후):**
    - 사용자가 선택을 완료하면, 선택한 버튼이 녹색(정답) 또는 빨간색(오답)으로 즉시 변경된다.
    - 화면 전체에 반투명 오버레이가 씌워지며, 중앙에 피드백 카드가 나타난다.
    - **(이미지 피드백):** 원본 이미지 위에 오류가 있는 부분이 원이나 사각형으로 강조 표시되며, "AI는 아직 신체 부위의 일관성을 표현하는 데 어려움을 겪습니다." 와 같은 구체적인 해설 텍스트를 제공한다.
    - **(텍스트 피드백):** 원본 텍스트에서 비논리적이거나 어색한 문장이 하이라이트 처리되며, "이 문장은 앞선 주장과 논리적으로 모순됩니다." 와 같은 해설을 제공한다.
    - 피드백 카드 하단에 `[다음 문제로]` 버튼이 있어 사용자가 직접 다음 단계로 넘어간다.

### [Screen 3.0] 성찰 질문 화면

- **진입 조건:** 3단계의 챌린지를 모두 완료한 후 진입한다.
- **UI/UX 및 기능:**
    - "학습 내용 돌아보기" 와 같은 제목을 표시한다.
    - 사용자가 완료한 챌린지 유형(이미지 또는 텍스트)에 따라 서로 다른 3개의 질문이 순차적으로 제시된다.
    - 각 질문은 3지 선다 객관식 형태로 제공된다.
    - 사용자가 오답을 선택하면, 선택지가 가볍게 흔들리거나 색상이 바뀌며 "정답이 아니에요. 다시 한번 생각해볼까요?" 와 같은 툴팁을 표시한다.
    - 사용자가 정답을 선택해야만 `[다음 질문]` 또는 `[결과 보기]` 버튼이 활성화된다. 3개의 질문에 모두 정답을 맞춰야 다음 화면으로 넘어갈 수 있다.

- **3.1 이미지 챌린지 완료 후 질문:**
    1.  **Q. AI가 생성한 이미지에서 우리가 주의 깊게 살펴봐야 할 '단서'로 가장 적절한 것은?**
        - [ ] 이미지의 해상도가 지나치게 높은 경우
        - [x] 인물의 손가락, 치아나 배경의 글씨처럼 복잡하고 일관성이 필요한 부분
        - [ ] 사진 속 인물이 너무 행복하게 웃고 있는 표정

    2.  **Q. AI 이미지 생성 기술의 가장 긍정적인 활용 사례는 무엇일까요?**
        - [ ] 역사적 사건을 왜곡 없이 완벽하게 사진으로 복원하는 것
        - [ ] 모든 전문 사진작가를 대체하여 비용을 절감하는 것
        - [x] 세상에 없는 독창적인 컨셉의 시안을 빠르게 만들어내는 것

    3.  **Q. 진짜 사진처럼 정교한 AI 이미지가 우리 사회에 미치는 가장 큰 우려는 무엇일까요?**
        - [ ] 고화질 이미지 때문에 인터넷 서버 용량이 부족해지는 문제
        - [x] 실존 인물의 얼굴을 합성한 '딥페이크'를 이용한 가짜뉴스 확산
        - [ ] 누구나 쉽게 예술 작품을 만들 수 있게 되어 예술의 가치가 하락하는 문제

- **3.2 텍스트 챌린지 완료 후 질문:**
    1.  **Q. AI가 생성한 텍스트의 논리가 가끔 어색한 가장 근본적인 이유는 무엇일까요?**
        - [x] AI는 문장의 의미를 '이해'하는 것이 아니라, 방대한 데이터의 통계적 패턴을 기반으로 단어를 예측하기 때문
        - [ ] AI가 일부러 사용자를 속이기 위해 거짓 정보를 섞어 넣기 때문
        - [ ] AI는 복잡하고 긴 문장을 만들어내는 능력이 부족하기 때문

    2.  **Q. 생성형 AI가 텍스트 관련 업무에서 가장 유용하게 사용될 수 있는 경우는?**
        - [ ] 최신 법률이나 의료 정보를 정확하게 요약하고 자문할 때
        - [x] 긴 회의록의 핵심 내용을 요약하거나, 이메일의 초안을 여러 버전으로 작성할 때
        - [ ] 한 사람의 일생에 대한 사실 기반의 자서전을 오차 없이 작성할 때

    3.  **Q. AI를 활용해 과제 보고서를 작성할 때, 우리가 가져야 할 가장 중요한 태도는?**
        - [ ] AI가 만든 결과는 항상 정확하므로 그대로 제출해도 무방하다.
        - [ ] AI가 사람보다 뛰어나므로 내 생각은 최대한 배제하는 것이 좋다.
        - [x] AI가 만든 내용은 반드시 사실 확인(Fact Check)을 거쳐야 하는 '초안'으로 여긴다.

### [Screen 4.0] 최종 결과 및 마무리 화면

- **UI/UX:**
    - 상단에 "챌린지 완료!" 와 같은 격려 문구를 표시한다.
    - 중앙 영역에 아래의 핵심 메시지를 텍스트 박스 형태로 강조하여 보여준다.
      > 이 웹앱을 통해 학생들은 생성형 AI에 대한 막연한 환상이나 공포에서 벗어나, 그 결과물을 비판적으로 수용하고 합리적으로 판단하는 **기초 소양(Literacy)**을 자기주도적으로 함양할 수 있을 것입니다. 또한, 지식 전달을 넘어 '생각하는 방법'을 훈련함으로써 다른 디지털 콘텐츠를 접할 때도 비판적 사고를 적용하는 태도를 기를 수 있습니다.
    - 하단에 2개의 버튼을 배치한다.
        - **버튼 A:** `[홈으로]` (Screen 1.0으로 이동)
        - **버튼 B (조건부):**
            - 이미지 챌린지를 완료한 경우: `[텍스트 챌린지 알아보기]` (Screen 1.0으로 이동)
            - 텍스트 챌린지를 완료한 경우: `[이미지 챌린지 알아보기]` (Screen 1.0으로 이동)

## 5. 기술 및 자산 관리 요구사항 (Technical & Asset Requirements)

- **플랫폼:** 웹 브라우저 기반의 반응형 웹앱
- **프레임워크:** (개발팀에서 선정, e.g., React, Vue.js)
- **반응형 디자인:** 모든 페이지는 모바일(360px)부터 데스크톱(1920px)까지 주요 화면 크기에서 UI가 깨지지 않고 자연스럽게 표시되어야 한다.
- **이미지 자산 관리:**
    - 프로젝트 루트에 `/image_sources/` 디렉토리를 생성한다.
    - 해당 디렉토리 내에 다음과 같이 하위 디렉토리를 구성하여 이미지를 관리한다.
        - `/image_sources/ai_images/`: 챌린지에 사용될 AI 생성 이미지 저장
        - `/image_sources/real_images/`: 챌린지에 사용될 실제 사진 이미지 저장
        - `/image_sources/ui/`: 앱의 아이콘, 로고 등 UI 관련 이미지 저장
